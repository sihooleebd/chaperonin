Computational Biology Orchestration

Wire tools.
Execute.

Chaperonin is a visual workflow engine for computational biology. Drag modules onto a canvas, wire them by data type, and run against a real Python backend — streamed live over WebSocket.

Open the App See how it works
01 — Visual & Text

Canvas and DSL,
always in sync.

Drag modules, wire connections, and the DSL updates live. Edit the text directly and the canvas redraws. Both are the same pipeline — choose whichever you think in. The dsl field is also what the backend receives — parse it directly or traverse the graph JSON, either works.

pipeline.chap
# Chaperonin DSL
 
scaffold = input(Structure.PDB, label="scaffold")
 
rfd_1 = RFDIFFUSION(
    pdb_file=scaffold,
    hotspot=???,
    length=100)
 
relaxed = ROSETTA_RELAX(
    structure=rfd_1.designed_pdb)
 
render = PYMOL(
    structure=relaxed.relaxed)
 
output(render.rendered, name="result")
02 — Type System

Connections that
know what they carry.

Every handle has a type. Structure.PDB satisfies an input declared as Structure — covariant by namespace. Union types accept multiple: Structure.PDB | Sequence.FASTA. Incompatible connections are rejected before the run message is ever sent.

Structure
.PDB .mmCIF

Compatibility
Structure.PDB → Structure   ✓ covariant
Structure.PDB → Sequence    ✗ mismatch
Structure.PDB → Structure | Seq… ✓ union
03 — Real-time Execution

Every event,
streamed live.

One persistent WebSocket. The backend topology-sorts the DAG and emits events as each node progresses — progress bars fill, log lines appear, nodes flip green as they complete. Cancel anytime; the backend acknowledges per node.

ws://localhost:8000/ws

WebSocket Protocol

One socket. Both directions.

Frontend sends a graph; backend streams events as execution progresses. Every run terminates with pipeline.done or pipeline.error — the frontend Promise hangs until one arrives.
